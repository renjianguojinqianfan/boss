# 微信小程序拍照签到功能技术方案

## 1. 功能需求分析

### 1.1 核心功能
- ✅ 相机拍摄功能：调用微信相机API进行拍照
- ✅ 工人选择功能：从全局工人列表中选择签到工人
- ✅ 位置获取功能：调用微信位置API获取当前位置
- ✅ 时间地点水印添加功能：在拍摄的照片上添加时间和地点水印
- ✅ 签到记录保存功能：将签到信息保存到本地存储

### 1.2 技术要求
- 稳定可靠：确保各功能模块稳定运行，处理各种异常情况
- 界面简洁易用：提供直观的用户操作界面
- 权限管理：合理处理相机和位置权限
- 数据持久化：使用本地存储保存数据

## 2. 技术方案设计

### 2.1 页面结构设计

#### 2.1.1 页面路径
- `pages/checkin/checkin` - 拍照签到主页面
- `pages/records/records` - 签到记录页面（新增）

#### 2.1.2 页面布局

**拍照签到页面 (checkin)**
1. 顶部导航栏：标题"拍照签到"
2. 工人选择区域：下拉选择框，显示工人列表
3. 相机预览区域：显示相机实时画面
4. 操作按钮区域：
   - 拍照按钮
   - 重新拍照按钮
   - 确认签到按钮
5. 照片预览区域：拍摄后显示照片，带水印
6. 位置信息显示：显示当前位置信息
7. 时间信息显示：显示当前时间

**签到记录页面 (records)**
1. 顶部导航栏：标题"签到记录"
2. 记录列表：显示所有签到记录
3. 记录详情：每条记录显示工人信息、时间、地点、照片

### 2.2 功能模块设计

#### 2.2.1 相机拍摄模块
- **API调用**：使用 `wx.createCameraContext()` 创建相机上下文
- **拍摄功能**：调用 `camera.takePhoto()` 方法进行拍照
- **预览功能**：拍摄后在页面上预览照片
- **错误处理**：处理相机初始化失败、拍摄失败等异常

#### 2.2.2 工人选择模块
- **数据来源**：从全局 `app.globalData.workers` 获取工人列表
- **选择方式**：使用 `picker` 组件实现下拉选择
- **默认值**：默认选择第一个工人
- **空状态处理**：当工人列表为空时，提示用户先添加工人

#### 2.2.3 位置获取模块
- **API调用**：使用 `wx.getLocation()` 获取当前位置
- **坐标转换**：使用 `wx.chooseLocation()` 获取详细地址信息
- **缓存策略**：缓存位置信息，避免频繁调用API
- **错误处理**：处理位置权限拒绝、定位失败等异常

#### 2.2.4 水印添加模块
- **实现方式**：使用 Canvas API 在照片上绘制水印
- **水印内容**：
  - 当前时间（年月日时分秒）
  - 当前地点（详细地址）
  - 工人姓名
- **水印样式**：半透明文字，合理布局，不遮挡主要内容
- **性能优化**：在后台线程处理水印添加，避免阻塞UI

#### 2.2.5 记录保存模块
- **数据结构**：
  ```javascript
  {
    id: "时间戳",
    workerId: "工人ID",
    workerName: "工人姓名",
    date: "日期",
    time: "时间",
    location: "位置信息",
    latitude: 纬度,
    longitude: 经度,
    photoUrl: "照片本地路径",
    timestamp: "ISO时间戳"
  }
  ```
- **保存方式**：调用 `app.addRecord()` 方法保存到本地存储
- **数据验证**：确保所有必填字段都有值
- **重复检查**：检查是否存在重复签到记录

### 2.3 数据流设计

1. **页面加载**：
   - 初始化相机上下文
   - 获取工人列表
   - 获取位置信息
   - 初始化页面数据

2. **拍照流程**：
   - 用户选择工人
   - 点击拍照按钮
   - 调用相机API拍照
   - 获取拍摄结果
   - 添加水印
   - 显示照片预览

3. **签到流程**：
   - 用户确认照片
   - 点击确认签到按钮
   - 收集所有信息
   - 验证数据完整性
   - 保存签到记录
   - 提示签到成功
   - 跳转到记录页面或返回首页

### 2.4 错误处理设计

#### 2.4.1 权限错误
- **相机权限**：当用户拒绝相机权限时，提示用户开启权限
- **位置权限**：当用户拒绝位置权限时，提示用户开启权限

#### 2.4.2 功能错误
- **相机初始化失败**：提示用户检查相机是否可用
- **拍照失败**：提示用户重新拍照
- **位置获取失败**：使用默认位置信息，提示用户
- **水印添加失败**：使用原图进行保存，提示用户

#### 2.4.3 数据错误
- **工人列表为空**：提示用户先添加工人
- **数据保存失败**：提示用户重试
- **重复签到**：提示用户该工人今日已签到

### 2.5 用户体验优化

1. **加载状态**：
   - 相机初始化时显示加载动画
   - 位置获取时显示加载动画
   - 保存记录时显示加载动画

2. **操作反馈**：
   - 拍照成功时显示成功提示
   - 签到成功时显示成功提示
   - 错误时显示错误提示

3. **界面设计**：
   - 简洁明了的操作界面
   - 大按钮设计，便于工地环境操作
   - 实时预览，确保拍摄效果

4. **性能优化**：
   - 延迟加载非关键资源
   - 优化相机预览性能
   - 缓存位置信息

## 5. 风险评估

### 5.1 潜在风险
1. **权限风险**：用户可能拒绝相机或位置权限
2. **性能风险**：相机预览和水印处理可能影响性能
3. **兼容性风险**：不同设备的相机和位置API可能存在差异
4. **数据风险**：本地存储可能被清除，导致数据丢失

### 5.2 缓解措施
1. **权限处理**：提供清晰的权限申请提示，引导用户开启权限
2. **性能优化**：优化相机参数，使用合适的图片尺寸，异步处理水印
3. **兼容性处理**：测试不同设备，处理API差异
4. **数据备份**：考虑定期导出数据或提供云备份功能
